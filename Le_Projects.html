Honors Digital Electronics (2022-2023)

Quiz Grader Circuit

Honors Digital Electronics

November 7, 2022 - November 30, 2022

Background: In this project, my group had the goal of designing a circuit and building a breadboard from an AND-OR-invert (AOI) logic statement modeled after a real-world system. Our group decided to break down the logic and build a circuit on a quiz grader program that allowed teachers to save time on grading. I was responsible for building the "half-credit" circuit, which was the most logically intensive circuit and required the most logic gates. I designed this circuit in the simulation program, Multisim, and breadboarded it. All in all, this project has taught me how to break down real-life situations into logic statements, use that statement to create a schematic on Multisim, and use the Multisim to design a breadboard in real life.

Greenfoot Quizlet Board Game

Computer Science Applications

July 18, 2021 - August 30, 2021

Game: Players will roll dice to move around the board. When one player lands on a question tile, a question will display and that player will need to press the 1, 2, 3, or 4 keys. The goal of the game is to land exactly on the finish line.

Background: The goal of this project was to create a two-player board game from scratch using the Greenfoot engine. The game includes has to include at least a 40-tile board with tiles that will display a question when a player lands on it. To win the game, the player has to land exactly at the finish line. This project taught me to use object-oriented code that is easily expandable and flexible. I also learned how to read forums and get help with code online.

Compound Machine Project

Principles of Engineering

November 9, 2020 - December 4, 2020

Background: The goal of this project was to create a Rube Goldberg machine that can pull up a 250-pound weight, 2 feet in the air. A Rube Goldberg machine is a complex series of mechanisms that fulfill a simple task. In this project, I was responsible for animating two sections of the machine in the Unity engine. We had to compromise and build our machine virtually since we realized that we did not own any of the materials required to construct a physical machine. I had to learn how to code in C# and quickly familiarize myself with the Unity interface in a limited time frame. Overall, this project was a lot of fun since I learned more about programming and the Unity engine.

Oh My Spikes!

Computer Science Principles

September 23, 2019 - October 8, 2019

Game: Balance the bird in the center of the screen and try to make it to the sides of the screen while simultaneously avoiding spikes, try to get the highest score!

Background: This project was completely built within the MIT App Inventor, a web-integrated Andriod development platform. Me and my partner, Dustin Sun, were tasked with the challenge of modifying an already-developed game within two weeks while recording and documenting every step along the way. The communication part proved to be a difficulty since the only way to cooperate is to constantly save and share that modified version. Sometimes when we forget to check the update log, we would end up working on an outdated version and have to manually rewrite our code on the updated version. Through these miscommunications, we learned to communicate better and to be more careful when we modify our code. Aside from improving my communication skills, it taught me about the different types of algorithms and how to make them.
