POE (2020-2021)

Upcycle Project

Principles of Engineering

February 10, 2021 - March 7,2021

Background: The goal for this project was to create a product made out of recyclable materials that can be sold in the market. Every year, 250 million tons of trash is thrown away by America alone. If we can find other ways to reuse plastic we can reduce the amount of trash polluting our oceans. Throughout this project, I learned how to set deadlines and manage a team of people. Moreover, this project has also reminded me of the pollution problem on our planet. If we continue to fill up our landfills at this rate; Eventually, we will run out of space to put our trash. For this reason, we must find new ways to limit our trash output and recycle when we can.

CSP (2019-2020)

Photo Borders

Computer Science Principles

March 20, 2020 - April 1, 2020

Background: This project was built using Python and the Pillow library. The goal of this project was to modify images to satisfy a client's requirements; our client tasked us with creating borders around images. At the beginning of the project, I had trouble coding with the Pillow library and had to constantly look up the documentation. Through this process though, I had familiarized myself with the documentation and got better at researching. Another lesson that I learned from this project was how useful modules and libraries are. They can save you time on a big project or even open up new possibilities for new projects, you can use them as long as you give credit to the creator.

Oh My Spikes!

Computer Science Principles

September 23, 2019 - October 8, 2019

Game: Balance the bird in the center of the screen and try to make it to the sides of the screen while simultaneously avoiding spikes, try to get the highest score!

Background: This project was completely built within the MIT App Inventor, a web integrated Andriod development platform. Me and my partner, Dustin Sun, were tasked with the challenge of modifying an already developed game within two weeks while recording and documenting every step along the way. The communication part proved to be a difficulty since the only way to cooperate is to constantly save and share that modified version. Sometimes when we forget to check the update log, we would end up working on an outdated version and have to manually rewrite our code on the updated version. Through these miscommunications, we learned to communicate better and to be more careful when we modify our code. Aside from improving my communication skills, it taught me about the different types of algorithms and how to make them.
